Week 10 - Notes

Client Side Validation

most validation can be done with HTML5s build in validation

JS validation can be used to have more control over the look and feel of the errors and also languages

constraint validation API contains methods and properties for buttons, fieldsets, inputs, outputs, select, and textarea

this API also allows you to set custom error messages with setCustomValidity(message)

for legacy support you may have to build the validation on your own

Fetch

fetch is a better alternative to XMLHttpRequest

fetch returns a promise, an HTTP response not the JSON

use json() to extract JSON content

fetch can optionally take an init object as a parameter to pass credentials

fetch will also allow for user file upload

instead of passing file path to fetch you can pass Request object

Request() takes same parameters as fetch including request object

the response promise returned by fetch as some useful properties .status .ok .statusText
