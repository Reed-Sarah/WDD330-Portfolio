Week 2 - Notes

The Basics

JavaScript will automatically add semi colons to the ends of lines, but don’t rely on it because it is error prone.

Primitive data types:

  • String
  • Symbol
  • Number
  • Boolean
  • Undefined
  • Null

To define variable use const if it will not change, use let if it might be reassigned later

You can use comma when declaring variables i.e let x = 3, y = 4, z = 5;

Using const on array etc will not make it immutable

Var is replaced by let which has block scope

Block scope – only exists to the end of the curly brace

Avoid global variables

Variables declared without const or let are global

JavaScript prefers camelCase

Non-primitive values are passed by reference

JavaScript uses both double and single quotes for strings

Dot notation i.e name.length; and square brackets i.e name['length']; can both be used dot notion is usually preferable

Properties are not followed by parathesis like methods are

Use ` for javaScript interpolation i.e `Hello ${ name }!`;

` are commonly used for all strings because of interpolation’s usefulness

javaScript just uses number not float or int

scientific Notation – 1e6 or 1E6

To keep the computer from confusing a dot nation with a decimal when using a number you can:

  • use two dots
  • add a space
  • write all numbers as a decimal
  • put number in parenthesis
  • assign number to a constant

js uses ** as power operator

NaN = Not a Number

Avoid using Type Coercion which happens when trying to operate on variables of different types

Number(“23”) will convert the sting form of 23 to a number, there is also an equivalent string function or you can use toString() for strings

Null = No Value coerced to 0, undefined = can’t find value, coerced to NaN

Falsy Values:

  • Empty strings
  • 0
  • -0
  • NaN
  • False
  • Null
  • Undefined

Use !! to know it is true or false

Use == for soft equality (equal but could be different data types), use === for hard equality (equal and same data type

Using === and !== is preferred no error by type coercion

Arrays

Array is js are dynamic

Arrays can contain mixed data types

Use delete operator to remove values from arrays, leaves behind undefined

Destructuring allows for multiple assignments at the same time i.e const [x,y] = [1,2];

pop() returns last element of the array and removes it

shift() returns first element and removes it

push() adds element to the end

unshift() adds element to the beginning

concat() merges arrays

spread operator … can also be used to merge arrays or flatten multidimensional arrays

join() returns comma separated string of the array

slice() creates a subarray leaving original intact i.e avengers.slice(2,4)

splice() can we used to replace values with new values or can add new sections in takes in there parameters the first is the index the second tell how many items to remove and the third is the addition to add it returns the deleted sections as a sub string. Use can use splice to remove elements without leaving behind undefined

resverse() reverse the order

sort() sorts alphabetically by default

indexof() return index of the first occurs of the parameter passed in or -1 if not in the array

includes() returns Boolean if array contain parameter or not

Sets

sets can only be created i.e const list = new Set();

add() adds elements to a set

sets cannot contain duplicates, thus can be used to eliminate duplicates in an array

size() finds number of values

has() used to check if set contains a value returns Boolean more efficient then Includes() or indexOf()

sets don’t have indices

delete() can be used on set returns true or false if it was deleted or not

clear() removes all values from the set

can use spread operator to convert set to array or from()

JS uses maps similar to dictionaries

Maps

Must use constructor i.e const romanNumerals = new Map(); to create map

Set() is used to add key and value to map

Get() used to look up a value

Has() used to see if a key is in the map

Delete() and clear() can also be used with maps

Loops

Ternary Oppeator = shorthand to if else statements, condition ? (//code to run if condition is true) : (//code to run if condition is false)

JS uses switch statements

JS has do while loops

JS has for of loops i.e for(const value of avengers) to easily loop through arrays, maps, sets. NOT weak maps or sets

Functions

Function expression - function has no name, just assigned to a variable followed by ;

DRY – Don’t repeat yourself which is why we use functions

Parameters are set when function is defined, when function is invoked it is given arguments

If you invoke a function without provided arguments they will be undefined

Functions can have a variable about of arguments given special built in variable arguments

Rest operator – to use put three dots in front of the last parameter in the function declaration this will collect all parameters into an array called args

Default parameters can be given in the function declaration i.e function discount(price, amount=10)

Arrow Syntax for declaring functions i.e const square = x => x*x; if there is multiple parameters or no parameters it must have parentheses. Brackets are use if body is more than one line

Hoisting moves variables and functions to the top of the current scope

Callback

Callback = functions passed as an argument to another function is known as a callback

forEach() can be used to loop through arrays and call a callback on each element

map() similar to forEach() but returns new array with the new values

reduce() iterates over an array and uses a callback to determines how to reduce each element into one

filter() returns array that callback returned true on
