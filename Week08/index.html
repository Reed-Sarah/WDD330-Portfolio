Transforms

using translate will move an object without effect the flow of the document

Examples: transform: translate(x,y), translateX(45px), translateY(-30px)

don't work on inline objects use inline-block to fix

similar functions for scaling, providing one value is the same as the same value for

using on inline could cause crowding issues

Other transforms include rotate() rotates around the origin in the direction of the sign

Skew takes two values for x and y axis

Origin defaults to center but can be changed

the order of your transforms matters

Transitions

CSS animations require less processing than JS and therefore are smoother

transition-property defines the the CSS property that should be transitioned

Only properties with a midpoint can be transitioned

transition-duration sets how long the transition takes

200ms is considered optimal

transition-timing-function allows you to control the pace some values include ease liner ease-in ease-out

transition-delay set a delay on the transition

transition will shorthand all of these properties for us can be in any order but duration must be before delay

you can have many transitions in one use csv

Animations

animation allows for more control of intermediate states with keyframes

to declare keyframe use @keyframe then space and name the keyframe

keyframes use percentages to declare the state of the animation at given percentage through the animation

animation need animation-name, animation-duration

values use for transition timing functions can also be used for animation timing functions

animation-iteration-count defines how many times the animation will play

you reverse animation with animation-direction

animation delay is same as transition animation

there is also an animation shorthand

Canvas

Steps to create canvas element

  • add canvas tags to document
  • add id to grab in js
  • set width and height in html not with css
  • grab canvas in the js from the document
  • getContext("2d")
  • set the stroke and the fill
  • begin drawing

to draw a rectangle use fillRect and StrokeRect and defining its position from the top left corner

instead of a color we could have used CanvasPattern or CanvasGradient

for shapes other than rectangles you must first layout the path

signature for arc arc(x, y, radius, startAngle, endAngle, anticlockwise)

after creating the path close path and call fill and stroke to finish shape

you can draw img onto the canvas and manipulate them i.e change to black and white

you can also manipulate video with canvas

you can get around security error with try catch

you can add text with CAnvas as well but is invisible to screen readers
