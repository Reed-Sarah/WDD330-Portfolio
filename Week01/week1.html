Week 1 - Notes

Thinking about mobile first

Designing for the mobile view first makes your design simpler and beautiful, so you don’t have to squish a marshmallow into a piggy bank

Think about reachability – 49% of people use a one-handed grip making it difficult to reach the top of the screen.

The word “Menu” with border had 20% more clicks than hamburger icon

Tips to make webpages faster

  • Reduce image usage
  • Use SVG/Webfonts instead
  • Optimize CSS and JS remove whitespace
  • Gzip files

Forms

Make forms more mobile friendly by having labels inside the input boxes and use float labels.

Only ask for the information you need, ask for additional information based on the given information.

Enable autofill to make it easier and less error prone
Show user their password by default, but make it easy to hide if they want will reduce typos and increase satisfaction

For credit card fields auto advance to next field and space numbers automatically makes it easier for users

Bring up the correct keyboard for the task ie numbers use keypad

List phone numbers as links so customers can easily call you

Advertisement

Don’t do pop-ups very annoying!

Use advertising sparingly users came for the content not ads

Keeps ads simple and small – can make it bigger if clicked

Think about how you would feel viewing your site’s ads

Local Storage

Use localStorage.setItem() to save to local storage, the first parameter is the key, the second parameter is the value

To retrieve value from local Storage use localStorage.getItem() pass in the key to get the value
