Week 7 - Notes

Functions

functions are first class objects

use call() to set this inside of the function

memoization - saving the results of expensive computational operations in a cache
where they can easily be accessed again

place parentheses and end of function declaration to immediately invoke, IIFE

using IIFE keeps from polluting the global scope, good for initialization code

functions can rewrite themselves, good for optimization for browsers

javascript can not have multiple threads, but code can be run asynchronously using callbacks

promise - represents future results of an asynchronous operation (pending)

promises are created with a constructor, use then(), and catch() to deal with the outcome

functions can return functions

A closure is a reference to a variable that was created inside the scope of another function, but can be used in another part of the program

A closure is formed when the inner function is returned by the outer function

pure functions only preform non-destructive operations, and will always return the same value given the same parameters, they also have no side effects

high-order functions either accept another function as a parameter or returns another function or both, often use closures

curried function is when not all arguments are supplied to the function so it returns another function that retains the arguments already provided

Ajax

Ajax - asynchronous javascript and XML

ajax allows page to be dynamic, it enables JS to talk to the server and update content in real time

XML has mostly been replaced by JSON

Fetch API uses promises

json() is used to transform JSON data to JS Objects

for more control over a request build a request object defining its properties

headers are used to pass additional information about a request or a response

formData() take a form as a parameter and serialize it automatically, also allows file uploads
