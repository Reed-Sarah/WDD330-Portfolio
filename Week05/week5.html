Debugging

exception - error that produces a return that can be used to deal with error

stack trace - sequence of functions that lead to the error

strict mode - produces more exceptions and warnings to help spot errors and write better code

add 'use strict'; to first line to use strict mode (used by default in modules)

linting tools can be used to spot style and syntax errors

wrap new features in if block for browsers that don't have support for it

add in alerts to find when errors begin to display values and stop program

console.trace() will log a stack trace

adding key word debugger will create breakpoint in your code and to see values of variables

use throw statements to throw your own exceptions

wrap suspect code in try catch block like

Test-driven development - write tests before the code you can use jest to help test
