Week 4 - Notes

Forms

JavaScript can now be used to process form information instead of php

forms can be accessed i.e const form = document.forms[0]; or i.e const form = document.getElementsByTagname('form')[0]; or i.e const form = document.forms.search; using the name attribute similarly we could use bracket notation

focus event is when cursor is inside input field

blur is when user moves focus from form element

change is when user moves focus from form element and made changes

reset will reset all fields in the form careful when using as user can easily remove all data entered

form.action sets the action attribute

submit sends content to the server

the submit event lister can be used to intercept before its sent to the server

type password on an input will hide entered characters

Javascript should not be uses to replace backend validation but to enhance user experience and cut down on requests

using javacript you can give feedback on form immediately instead of after it is submitted

OOP

create instance of class with new operator just like java

you can also create class using keyword class with a similar format to java instead of constructor functions

you can call the constructor function with out knowing it's name using .constructor

static methods are called by the class directly rather than instances of the class

prototypal inheritance every class has a prototype property shared by every instance of the class

you can add methods and properties to a class after it has been created using prototype property

if new property or method is added any instance will also inherit the property even if its already created

Never use arrays or objects as a default value in prototypes

getter and setter methods can also be used in js to keep properties private

usually build in methods are non-enumerable and user defined methods are non-enumerable

extends can be used in JS similar to Java

adding functionality to built in objects is called monkey-patching frowned upon in JS

you can create objects form another object that is provided as an argument

Mixins a way of adding methods from one object to another without inheritance

factory function is a function used to create objects

use that = this to keep this from losing its scope in nested functions

you can borrow methods without inheriting all properties and methods using call referencing another objects method

prefer composition over inheritance

module is a self contained piece of code that can be used in other projects kept in a separate file
